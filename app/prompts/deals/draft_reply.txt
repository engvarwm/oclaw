Задача: реорганизовать проект OpenClaw-адаптера Hub в модульную структуру с разделением routers, services, schemas, prompts, tests.

Контекст:

* Сейчас OpenClaw-адаптер находится на сервере в `/opt/hub`.
* Основной файл сейчас: `/opt/hub/main.py`.
* Это отдельный FastAPI-адаптер между CRM Hub и OpenClaw/OpenAI.
* Nginx проксирует:

  * внешний URL `https://engvar.ee/hub/...`
  * во внутренний adapter `http://127.0.0.1:18089/...`
* Уже есть рабочие endpoints:

  * `GET /health`
  * `POST /deals/draft-reply`
  * `POST /deals/next-step`
* Важно: внешние URL менять нельзя.
* В будущем будут добавляться запросы по разным модулям Hub:

  * Deals / Сделки
  * Emails
  * Calls / телефония
  * Tasks
  * Finance / банки
  * Taxes
  * Plan-Fact
  * Repairs
  * другие модули
* Нужно подготовить архитектуру, чтобы новые endpoints не складывались в один огромный `main.py`.

Цель:
Сделать нормальную структуру проекта:

```text
/opt/hub
  main.py
  app/
    __init__.py
    main.py
    core/
      __init__.py
      config.py
      security.py
      logging.py
      errors.py
    routers/
      __init__.py
      health.py
      deals.py
    services/
      __init__.py
      openclaw_client.py
      prompt_renderer.py
      response_parser.py
      deals/
        __init__.py
        draft_reply_service.py
        next_step_service.py
    schemas/
      __init__.py
      common.py
      deals.py
    prompts/
      deals/
        draft_reply.txt
        next_step.txt
    tests/
      __init__.py
      conftest.py
      test_health.py
      test_deals_draft_reply.py
      test_deals_next_step.py
```

Можно адаптировать структуру под текущий проект, но обязательно разделить:

* FastAPI app;
* routers;
* services;
* schemas;
* prompt templates;
* OpenClaw client;
* tests.

1. Совместимость

Нельзя ломать существующие endpoints:

```text
GET /health
POST /deals/draft-reply
POST /deals/next-step
```

Они должны работать после рефакторинга точно так же.

Форматы ответов сохранить:

* `/health` возвращает `{"ok": true, "service": "..."}`
* `/deals/draft-reply` возвращает:

```json
{
  "ok": true,
  "draft_reply": "..."
}
```

* `/deals/next-step` возвращает:

```json
{
  "ok": true,
  "recommendation": {...}
}
```

2. main.py

Существующий `/opt/hub/main.py` сделать минимальным compatibility entrypoint:

```python
from app.main import app
```

Вся логика должна переехать в `app/main.py`, routers и services.

3. app/main.py

Создать FastAPI app:

* подключить routers:

  * health router;
  * deals router;
* настроить exception handlers;
* настроить logging;
* не хранить бизнес-логику в `app/main.py`.

Пример:

```python
from fastapi import FastAPI
from app.routers import health, deals

app = FastAPI(title="Hub OpenClaw Adapter")

app.include_router(health.router)
app.include_router(deals.router, prefix="/deals")
```

4. Config

Создать `app/core/config.py`.

Вынести туда:

* `HUB_OPENCLAW_TOKEN`
* `OPENCLAW_BIN`
* timeout
* session keys
* любые future env.

Пример:

```python
from pydantic import BaseModel
import os

class Settings(BaseModel):
    hub_openclaw_token: str = os.getenv("HUB_OPENCLAW_TOKEN", "")
    openclaw_bin: str = os.getenv("OPENCLAW_BIN", "openclaw")
    openclaw_timeout_seconds: int = int(os.getenv("OPENCLAW_TIMEOUT_SECONDS", "180"))

settings = Settings()
```

Не печатать secrets в логах.

5. Security

Создать `app/core/security.py`.

Вынести проверку Bearer token:

```python
def verify_hub_token(authorization: str | None) -> None:
    ...
```

Требования:

* если токен не настроен — 500;
* если authorization отсутствует — 401;
* если Bearer token неверный — 401;
* не логировать токен.

Все protected routes должны использовать эту функцию.

6. OpenClaw client

Создать `app/services/openclaw_client.py`.

Вынести запуск OpenClaw CLI:

```python
def run_openclaw_agent(
    *,
    session_key: str,
    prompt: str,
    timeout: int | None = None,
) -> dict:
    ...
```

Внутри:

* использовать `subprocess.run`;
* запускать:

```bash
openclaw agent --session-key ... --message ... --json --timeout ...
```

* возвращать parsed JSON;
* если OpenClaw вернул ошибку — бросать понятное исключение;
* если JSON не разобрался — бросать понятное исключение;
* логировать только краткую ошибку, без полного prompt и без secrets.

16. Проверка после рефакторинга

Выполнить:

```bash
python -m compileall app main.py
```

Запустить локально:

```bash
uvicorn app.main:app --host 127.0.0.1 --port 18089
```

Проверить:

```bash
curl -i http://127.0.0.1:18089/health
```

После systemd:

```bash
sudo systemctl restart hub-openclaw-adapter
sudo systemctl status hub-openclaw-adapter --no-pager
```

Проверить внешне:

```bash
curl -i https://engvar.ee/hub/health
```

17. systemd

Проверить service:

```text
ExecStart=/opt/hub/.venv/bin/uvicorn main:app --host 127.0.0.1 --port 18089
```

Так как `/opt/hub/main.py` остаётся compatibility entrypoint, service можно не менять.

Если решишь менять service на:

```text
ExecStart=/opt/hub/.venv/bin/uvicorn app.main:app --host 127.0.0.1 --port 18089
```

то обязательно указать это отдельно и проверить restart.

Предпочтительно пока НЕ менять systemd, чтобы снизить риск.
